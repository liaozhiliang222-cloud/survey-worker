# 调研工具 PWA 开发计划

## 产品定位

调研项目效率工具箱，不优先替代 Excel、SPSS 或问卷平台的数据分析能力，而是优先解决研究员在项目执行、模型专项分析、交付前质检中的细碎刚需。

## MVP 已落地方向

1. 工具箱首页：用“调研前 / 调研中 / 调研后 + AI 增值”的方式组织功能。
2. 样本量计算器：支持置信水平、误差、用户规模、分群数量、有效回收率。
3. 配额设计器：支持单一配额和交叉配额，按比例生成目标样本量。
4. 问卷上线质检助手：支持粘贴问卷稿，检查题号、跳题引用、随机轮换、重复选项和细节错误。
5. 清洗规则生成器：基于问卷稿生成时长、甄别、陷阱题、直线题、开放题等规则建议。
6. 表头设计助手：基于问卷稿推荐分群表头和不建议入表题。
7. PSM 价格敏感度分析：支持四列价格数据粘贴，生成关键价格点和价格曲线表。
8. 答题时间估算：按题型、选项数、矩阵题和开放题估算问卷完成时长。
9. 质检报告导出：把问卷上线质检结果导出为 Markdown 报告。
10. KANO 模型分析：支持属性分类人数导入，输出属性分类、Better-Worse 系数和 CSV。
11. MaxDiff：支持题组设计模板生成和 Best-Worst 简单计分。
12. ABC 用户价值模型：支持粘贴问卷后识别态度、行为、消费三类指数候选指标。
13. 专项模型入口：先明确 KANO、MaxDiff、ABC 的开发顺序。

## 下一阶段优先级

### P0：把调研前工具做完整

- 配额设计器：城市、性别、年龄等维度自动生成单一或交叉配额表。（已支持多维度单一配额与交叉配额）
- 答题时间估算：按题型和题量估算问卷时长。（已完成第一版）
- 质检报告导出：把调研前审查结果导出为 Markdown 或 PDF。（已支持 Markdown）

### P1：做出差异化模型工具

- PSM：导入四个价格问题，输出四条累计曲线和关键价格点。
- KANO：导入正反向题，输出属性分类和 Better-Worse 图。（已完成汇总计数版）
- ABC 用户价值模型：识别态度、行为、消费指数候选指标。（已完成 1.0）
- 模型结果导出：支持 PNG、CSV 和报告摘要。（PSM/KANO 已支持 CSV）

### P2：扩展专业壁垒

- MaxDiff 设计模板。（已完成第一版）
- MaxDiff 简单计分。（已完成第一版）
- 问卷逻辑检查助手。（已增强条件跳题来源、选项、目标检查）
- 清洗规则配置：时长阈值、陷阱题、直线题、开放题字数等规则模板。（已支持时长、直线题、开放题阈值）
- 表头设计：规划分群表头结构，识别不建议入表题，减少后期制表沟通成本。

### P3：加入 AI 增值能力

- AI 问卷措辞审查。（已完成提示词工作台第一版）
- AI 问卷逻辑风险提示。（已完成本地预检摘要 + AI 提示词）
- AI 解释 PSM/KANO/MaxDiff 结果。
- AI 生成调研前质检报告摘要。（已完成 AI 摘要第一版）

## 技术策略

当前版本使用零依赖静态 PWA，方便快速验证产品方向。等核心交互稳定后，再升级为 React/Vite/TypeScript 项目，并引入图表、文件导入和导出能力。

## 2.0 版本开发计划

### 2.0 产品目标

2.0 不建议简单堆更多小工具，而是把 1.0 的工具箱升级为“调研项目工作台”：围绕一个具体项目，把问卷设计、上线质检、配额规划、清洗规则、专项模型、报告摘要串起来，减少研究员在 Excel、问卷平台、文档和聊天工具之间反复复制粘贴。

核心变化：

1. 从“单工具使用”升级为“项目流使用”。
2. 从“人工粘贴后即时输出”升级为“可保存、可复用、可追踪”。
3. 从“规则型检查”升级为“规则 + AI 辅助解释 + 报告化输出”。
4. 从“模型计算结果”升级为“研究结论草稿和交付材料雏形”。

### 2.0 信息架构建议

保留当前“调研前 / 调研中 / 调研后 / AI 增值”的导航，但在顶部增加“项目”概念：

- 项目首页：项目名称、研究类型、问卷稿、样本目标、当前阶段、最近输出。
- 调研前：样本量、配额设计、答题时间、上线质检。
- 调研中：清洗规则、表头设计、回收过程检查模板。
- 调研后：PSM、KANO、MaxDiff、ABC、模型解释与导出。
- AI 助手：作为跨阶段浮层或侧栏，而不是只放在调研后。

### P0：项目化与保存能力

目标：让工具从一次性页面变成可持续使用的项目工作台。

1. 项目档案
   - 支持创建项目：项目名、客户/品牌、研究类型、样本目标、研究阶段。
   - 支持保存问卷稿、配额方案、质检结果、清洗规则、模型结果。
   - 本地优先，可先使用浏览器本地存储；后续再考虑后端账户体系。
   - 进度：已启动第一版，支持总览页填写项目档案、本地保存、恢复和同步问卷稿到多个工具。

2. 项目总览页
   - 展示项目完成度：问卷已质检、配额已设计、清洗规则已生成、模型是否已完成。
   - 展示关键提醒：样本量不足、配额比例不满 100%、跳题目标缺失、模型数据异常。
   - 进度：已启动第一版，支持项目完成度、检查清单和建议下一步。

3. 输出记录管理
   - 每次质检、清洗规则、模型分析形成一条记录。
   - 支持重新打开、复制、导出。
   - 避免用户每次刷新后结果丢失。

### P1：文件导入与统一解析

目标：降低复制粘贴成本，让真实业务材料更容易进入工具。

1. 问卷稿导入
   - 支持 `.docx`、`.xlsx`、`.md`、`.txt`、`.csv` 的问卷稿导入。
   - 自动识别题号、题干、选项、跳题说明、随机说明、矩阵题。
   - 形成统一的问卷结构数据，供上线质检、清洗规则、表头设计、ABC 指标识别共用。
   - 进度：已完成 DOCX / XLSX / Markdown / TXT / CSV 直接导入第一版，XLSX 支持读取多个工作表，支持导入流程说明、识别预览和同步到项目总览。

2. 数据表导入
   - 支持 CSV / Excel 粘贴或上传。
   - PSM、KANO、MaxDiff 不再只依赖文本框粘贴。
   - 自动识别列名，用户确认映射关系。

3. 解析预览
   - 任何导入内容先展示“工具识别成了什么”。
   - 用户可手动修正题型、题号、变量名、选项类型。
   - 这是后续 AI 和自动报告的基础。

### P1：问卷上线质检 2.0

目标：从“发现明显问题”升级为“上线前审查清单”。

1. 规则库分层
   - 严重问题：跳题目标不存在、题号重复、选项缺失、排他项未说明。
   - 重要问题：随机/轮换规则不清、其他项缺少说明、矩阵题过长。
   - 建议问题：措辞不统一、选项颗粒度不一致、可能存在错别字。

2. 研究场景模板
   - 概念测试模板。
   - U&A 模板。
   - 满意度/NPS 模板。
   - 价格研究/PSM 模板。
   - KANO/MaxDiff 专项模板。
   - 进度：已完成第一版场景选择，支持按场景追加专项缺漏提醒，并写入 Markdown / HTML 质检报告。

3. 质检报告升级
   - 导出 Markdown + HTML。
   - 报告结构包括：总体结论、上线阻断项、建议优化项、逐题问题清单。
   - 支持“给问卷平台测试人员看的版本”和“给客户/项目经理看的版本”。
   - 进度：已完成 Markdown + HTML 报告导出第一版，HTML 报告包含质检概览、上线阻断项、问题明细和复核建议。

### P1：配额与样本管理 2.0

目标：让配额工具更贴近项目执行，而不是只做比例计算。

1. 配额模板
   - 常用维度模板：性别、年龄、城市级别、区域、用户类型、品牌使用状态。
   - 支持一键添加常见选项。
   - 支持维度复用到不同项目。

2. 配额校验
   - 单一配额比例必须等于 100%。
   - 交叉配额总样本量必须等于目标样本量。
   - 自动提示空单元格、极小样本格、不可执行配额。

3. 回收监控模板
   - 2.0 先不做实时平台对接。
   - 支持粘贴当前回收数，自动对比目标配额，输出缺口和超收提醒。
   - 适合项目执行中每天追踪配额进度。

### P1：清洗规则与表头设计 2.0

目标：把调研中后期最耗沟通的部分提前标准化。

1. 清洗规则可编辑
   - 自动生成规则后，用户可以启用/停用、修改阈值、添加备注。
   - 规则类型包括：答题时长、甄别题、陷阱题、直线题、开放题质量、逻辑矛盾。

2. 清洗规则导出
   - 导出为 Excel/CSV/Markdown。
   - 字段包括：规则名称、适用题目、判断条件、处理建议、优先级。

3. 表头设计增强
   - 自动识别适合入表的分群变量。
   - 区分主表头、补充表头、不建议入表题。
   - 输出给数据处理同事可直接使用的表头说明。

4. 交叉表分析
   - 支持粘贴 CSV / Excel 复制数据，选择行变量和列变量。
   - 输出频数、列百分比、卡方检验、p 值和结果解读。
   - 进度：已完成 1.0 页面和 CSV 导出。
   - 进度：已新增“生成全部交叉表”模式，可自动识别单选、多选、量表、矩阵、排序、开放题和 NPS，并直接导出 Excel。

5. 数据加权
   - 支持 RIM 边际加权，用原始样本数据和目标边际结构迭代计算样本权重。
   - 支持 Cell 单元格加权，用当前样本和目标样本直接计算单元格权重。
   - 输出权重范围、有效样本量、边际校准结果和 CSV 权重表。
   - 进度：已完成 1.0 页面和 CSV 导出。

### P2：专项模型 2.0

目标：让模型不只是算数，而是能解释、导出、形成报告素材。

1. PSM 2.0
   - 支持 Excel/CSV 导入。
   - 支持异常样本识别：四个价格顺序不合理、极端值、缺失值。
   - 图表支持导出 PNG。
   - 输出价格区间解释和业务建议草稿。
   - 进度：已完成 SVG 图表导出 PNG 第一版。

2. KANO 2.0
   - 支持从正反向原始回答自动计算分类，而不只依赖汇总人数。
   - Better-Worse 图支持 PNG 导出。
   - 输出属性优先级建议：必保项、优化项、惊喜项、低投入项。
   - 进度：已完成 Better-Worse 图导出 PNG 第一版。

3. MaxDiff 2.0
   - 设计部分增加均衡度检查：展示次数、同现次数、位置分布。
   - 计分部分支持从原始选择数据汇总。
   - 结果输出偏好排序、Top 项目解释和问卷设计提醒。
   - 进度：已完成简单计分图表与 PNG 导出第一版。

4. ABC 用户价值模型 2.0
   - 从“识别候选指标”升级为“指标配置器”。
   - 支持用户确认 A/B/C 指标、设置方向、设置权重。
   - 支持导入分群数据后计算 ABC 得分。
   - 输出高价值用户画像摘要。

### P2：AI 助手 2.0

目标：AI 不单独作为一个页面，而是嵌入每个关键流程。

1. AI 侧栏
   - 在问卷质检、清洗规则、模型结果页面都可以打开。
   - 自动带入当前页面上下文。
   - 输出“解释 / 改写 / 总结 / 生成报告段落”。
   - 进度：当前页面已调整为待开发预留入口，避免误导用户以为已有正式 AI 能力。

2. AI 问卷审查
   - 识别复杂措辞、诱导性表述、选项不互斥、题目过长。
   - 给出可直接替换的问卷文案。

3. AI 模型解释
   - 将 PSM/KANO/MaxDiff/ABC 结果转换成研究员报告语言。
   - 输出：一句话结论、关键发现、业务建议、风险提醒。

4. AI 安全策略
   - 前端不暴露 API Key。
   - 2.0 若接入真实 AI，需要增加后端代理。
   - 支持用户确认是否发送问卷和数据到 AI。

### P3：导出与交付 2.0

目标：让工具输出能直接进入项目交付链路。

1. 统一导出中心
   - 当前项目的质检报告、配额表、清洗规则、表头设计、模型结果集中导出。
   - 支持 Markdown、CSV、HTML，后续支持 PPTX。

2. 报告摘要
   - 自动生成项目级摘要。
   - 包括：样本设计、问卷风险、清洗建议、模型发现。

3. 图表导出
   - PSM、KANO、MaxDiff、ABC 结果图支持 PNG。
   - 图表样式统一，便于复制进报告。

### 技术升级建议

2.0 可以继续保持“轻量、可本地打开”的优势，但建议逐步做结构升级：

1. 短期
   - 仍使用静态 PWA。
   - 增加本地存储、文件导入、统一数据结构。
   - 把过长的 `app.js` 拆成模块化文件。

2. 中期
   - 升级到 Vite + TypeScript。
   - 引入轻量图表库或继续使用 SVG，但封装图表组件。
   - 建立统一的问卷解析器、模型计算器、导出器。

3. 长期
   - 增加后端服务，用于 AI 代理、项目同步、团队协作。
   - 支持账户、项目共享、版本历史。
   - 与问卷平台或数据处理流程做接口级集成。

### 推荐开发顺序

1. 项目保存与项目总览。
2. 问卷结构化解析器。
3. 上线质检 2.0 与报告导出升级。
4. 清洗规则和表头设计可编辑化。
5. 配额回收监控模板。
6. PSM/KANO/MaxDiff/ABC 的文件导入与图表导出。
7. ABC 指标配置器和得分计算。
8. AI 侧栏与模型解释。
9. 统一导出中心。

### 2.0 验收标准

1. 用户可以创建一个项目，并在项目内保存问卷、配额、质检、清洗、模型结果。
2. 用户导入一份问卷稿后，至少 3 个工具可以复用同一份问卷结构。
3. 问卷上线质检报告可以直接发给项目组讨论。
4. 配额工具不仅能生成目标表，还能根据当前回收数输出缺口。
5. PSM/KANO 至少支持图表 PNG 导出。
6. ABC 模型可以从候选指标识别升级到用户确认指标配置。
7. AI 能基于当前页面结果生成可复制的报告段落。
